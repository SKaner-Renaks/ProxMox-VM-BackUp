# ProxMox-VM-BackUp (SSH)

Скрипт для резервного копирования архивных копий виртуальных машин Proxmox на локальный диск по протоколу SSH/SFTP.

## Функционал
- Автоматический поиск директорий бэкапов на сервере Proxmox.
- Копирование последних версий бэкапов для выбранных VM.
- Проверка существующих копий по размеру, дате и времени.
- Ротация локальных копий (индивидуальная настройка для каждой VM).
- Поддержка нескольких серверов.
- Предварительная проверка свободного места на диске.
- Информационная сводка перед началом копирования.

## Конфигурация (vm_config)
Параметр `vm_config` в файле конфигурации определяет поведение для каждой VM:
- **Положительное число (1, 2, 3...)**: Количество хранимых локальных копий. Скрипт скачивает последнюю копию и удаляет старые, если их больше указанного числа.
- **0**: Пропустить VM. Скрипт полностью игнорирует эту VM.
- **-1**: Очистка. Скрипт удалит все локальные копии этой VM и не будет скачивать новые.

## Глобальные настройки
- `TIMEOUT`: Таймаут сетевых операций (сек).
- `MIN_FREE_COPIES_WARNING`: Минимальное количество полных копий, при котором индикатор места остается зеленым (по умолчанию 5).

## Цветовая индикация (CLI)
| Цвет | Назначение |
| :--- | :--- |
| **GREEN** | Успех, OK, итоговая статистика |
| **BRIGHT_GREEN** | Достаточно места (>= 5 копий), позитивные уведомления |
| **YELLOW** | Предупреждения, ротация, имена файлов, пропущенные VM |
| **ORANGE** | Мало места (< 5 копий), предупреждения о месте |
| **RED** | Ошибки, критическая нехватка места |
| **CYAN** | Заголовки, прогресс-бары, спиннеры |
| **WHITE** | Значения переменных, даты, IP |
| **BOLD** | Выделение важных цифр и объемов |

## Использование
1. Запустите скрипт `pve_ssh_backup_vm.py` для генерации файла конфигурации.
2. Заполните `pve_ssh_backup_vm_config.py` данными ваших серверов и VM.
3. Запустите скрипт снова для выполнения бэкапа.

## Зависимости
- `paramiko` — библиотека для работы с SSHv2 протоколом. Используется для подключения к серверам Proxmox и передачи файлов по SFTP.
