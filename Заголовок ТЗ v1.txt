Техническое задание: Заголовок и стандарты разработки скрипта

1. Стек и Окружение
* Язык программирования: Python
* Версия: Python 3.14.4 (с обеспечением базовой обратной совместимости)
* ОС запуска (Хост): Windows Server Core 2025
* Удаленный сервер: Debian GNU/Linux 12 (bookworm)
* Платформа удаленного сервера: Proxmox VE 8.4.14 Virtual Environment
* Протокол доступа: SSH по ключам (авторизация настроена, ключи доступны)

2. Стандарты кода и Локализация
* Кодировка: UTF-8 с полной поддержкой кириллицы.
* Язык комментариев: Строго русский язык.
* Стиль форматирования: Соответствие стандарту PEP 8.
* Обработка путей: Все локальные пути ОС Windows внутри скрипта обрабатывать исключительно через встроенную библиотеку pathlib (объекты Path).

3. Версионность и Документация
* Старт версионности: v 0.01.00
* Правила изменения версий:
  * Дополнение / изменение / удаление функционала: +0.01.00 к текущей версии. Обязательная фиксация изменений с указанием версии в файле history.md.
  * Любая правка ошибки / корректировка: +0.00.01 к текущей версии. Обязательная фиксация изменений с указанием версии в файлах history.md и trouble.md.

* Файловая структура документации:
 * readme.md — На русском языке. Описание назначения скрипта, инструкция по использованию. Обязателен блок со списком внешних библиотек и кратким описанием, зачем они нужны.
 * history.md — На русском языке. Сохранение истории всех изменений скрипта с привязкой к версиям.
 * trouble.md — На русском языке. История найденных проблем в коде. Строго указывать версию, в которой найдена ошибка, описание проблемы, как исправлена, и версию патча, содержащую исправление.
 * requirements.txt — Стандартный файл зависимостей для пакетного менеджера pip.

4. Конфигурация и Логирование
* Файл конфигурации: Использовать отдельный файл в формате .py (например, config.py), расположенный в корневой папке скрипта. Имя файла конфигурации должно совпадать с именем основного скрипта.

* Логика первого запуска:
  * Если файл конфигурации отсутствует, скрипт должен автоматически создать его с дефолтными полями, примерами заполнения и комментариями.
  * После создания файла скрипт выводит в консоль сообщение: "Файл конфигурации [Имя].py не найден и был успешно создан. Пожалуйста, заполните его перед следующим запуском." и корректно завершает работу. Основное тело скрипта при этом не выполняется.
  * Чтение конфигурации: Переменные пользователя выносятся во внешний файл и импортируются стандартным способом Python: import filename (или динамическим импортом).

* Содержимое конфигурации: Все параметры, задаваемые пользователем (IP-адрес/имя удаленного сервера, логин, пути к локальным SSH-ключам и т.д.).

* Логирование (LOG): Настоящее логирование в файл на данном этапе не требуется. В коде необходимо предварительно подготовить пустые функции-заглушки для последующей реализации логирования.

5. Интерфейс пользователя (UI) и Консоль
* Стартовые правила: При запуске скрипт всегда выводит в консоль своё название, текущую версию, а также дату и время запуска в формате DD-MM-YYYY HH:MM:SS.
* Финальные правила: При завершении работы скрипт выводит дату и время завершения, а также общее чистое время работы (длительность выполнения).
* Визуализация: Обязательная поддержка цветов ANSI Color Codes для форматирования вывода в CLI/консоли.

* Режим отладки: До тех пор, пока нет прямого указания на скрытие вывода, скрипт транслирует в консоль подробный пошаговый ход выполнения (что делает скрипт в данный момент) для наблюдения и отладки.
* Режимы тестирования: Специализированные режимы test, dry-run или show не требуются.

6. Исключения и Безопасность
* Прерывание работы: Скрипт должен корректно перехватывать и обрабатывать ручное прерывание пользователем с помощью комбинаций Ctrl+C и/или клавиши F10.
* Освобождение ресурсов: При любом типе завершения (успешное, аварийное падение, перехват прерывания) скрипт обязан выполнить закрытие всех открытых SSH-сессий, сетевых сокетов и хэндлов файлов внутри блока finally.
* Сетевая стабильность: Абсолютно все сетевые операции (SSH-сессии, команды на Proxmox, запросы к API) должны иметь жестко заданный таймаут работы (timeout), чтобы исключить бесконечное зависание скрипта в случае сетевого сбоя.

7. Структура файла скрипта
* Скрипт представляет собой один файл, визуально разделенный на логические блоки с помощью разделителя # ===================================:
* Блок заголовок: Метаданные, авторство, версия.
* Блок переменных: Импорт библиотек, инициализация служебных переменных и констант, значения которых не задаются пользователем в конфигурации.
* Блок функций: Отдельные изолированные функции. Перед каждой функцией должны идти ровно две пустые строки (визуальный разделитель) и краткий однострочный комментарий, описывающий её функционал.
* Точка входа: Функция main() и стандартная конструкция проверки if __name__ == '__main__':.
